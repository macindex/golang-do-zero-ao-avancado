1. A Regra de Ouro da Formatação
Em Go, o compilador é extremamente rigoroso com a posição do else. Ele deve começar na mesma linha em que a chave de fechamento } do bloco anterior termina.

Aqui está uma explicação aprofundada sobre o tópico:

1. A Regra de Ouro da Formatação
Em Go, o compilador é extremamente rigoroso com a posição do else. Ele deve começar na mesma linha em que a chave de fechamento } do bloco anterior termina.

Se você colocar o else na linha de baixo, o compilador inserirá um ponto e vírgula invisível após a chave } e seu código não compilará.

Correto:

if condicao {
    // ...
} else {
    // ...
}

Errado:
if condicao {
    // ...
} 
else { // Erro de compilação!
    // ...
}

2. Else If (Condições Encadeadas)
O else if permite testar múltiplas condições em sequência. 
Assim como o else simples, ele deve seguir a regra da mesma
linha.

func categorizar(n int) string {
    if n < 0 {
        return "Negativo"
    } else if n == 0 {
        return "Zero"
    } else {
        return "Positivo"
    }
}

3. Escopo e o Bloco de Inicialização
Lembra-se que o Go permite inicializar variáveis dentro 
do if? O ponto crucial aqui é que essas variáveis também 
estão disponíveis dentro do bloco else e else if.

if valor, err := realizarCalculo(); err != nil {
    // 'valor' e 'err' estão disponíveis aqui
    fmt.Println("Erro:", err)
} else {
    // 'valor' e 'err' AINDA estão disponíveis aqui!
    fmt.Println("Resultado:", valor)
}
// Fora daqui, 'valor' e 'err' deixam de existir.

4. A Filosofia "Menos é Mais": Evitando o Else
Embora o else exista, a comunidade Go e os criadores da 
linguagem incentivam a sua omissão sempre que possível, 
utilizando uma técnica chamada Early Return (Retorno Antecipado).

A ideia é: se você encontrar um erro ou uma condição de 
saída, retorne imediatamente. 
Isso evita que o "caminho feliz" (o fluxo principal do código) 
fique indentado dentro de blocos else.

5. Quando realmente usar o Else?
O else deve ser usado quando você tem dois caminhos de 
execução que são mutuamente exclusivos e ambos têm peso 
similar na lógica do negócio, não sendo apenas um tratamento 
de erro ou uma validação inicial.

    var eCarro bool = false
    var valorDoAutomovel = 1000

    if eCarro {
        valorDoAutomovel = valorDoAutomovel + 55.5
    }

    if !eCarro {
        valorDoAutomovel = valorDoAutomovel + 20.5
    }

    fmt.Println("Valor final: ", valorDoAutomovel)
}
